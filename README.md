三个数据集的处理逻辑：

静态数据：作为基础特征，需要清洗和标准化

文本数据：转化为可用的文本特征（用于后续NLP模型）

时序数据：降维提取统计特征（避免维度爆炸）

二、各数据集清洗思路
1. 静态数据清洗

① 目标变量处理

移除 icu_los_hours ≤ 0 的记录（无效数据）

移除 icu_los_hours > 720 小时（>30天）的极端值（临床罕见，可能是异常记录）

② 分类变量处理

填充为 'Unknown' 而非删除（保留样本，避免数据丢失）

原因：ICU中缺失记录也有临床意义（可能未记录或未发生）

③ 数值变量处理

使用中位数填充（对异常值更鲁棒）

不用均值（避免极端值影响）

④ 为何不移除缺失率高的列

保留所有特征，让模型自行判断重要性

后续建模时可使用特征选择或正则化

2. 文本数据清洗

① 处理缺失值

先填充空字符串（fillna('')）避免聚合时报错

② 合并文本

按 stay_id 分组，将同一患者的多个放射学报告合并

用空格连接（保留原始语义）

③ 提取文本特征

添加 text_length 作为简单特征

为后续更复杂的文本特征提取（TF-IDF、Word2Vec、BERT等）做准备

设计考虑：

不删除无文本的患者（填充空字符串）

保留原始文本内容供后续深度模型使用

3. 时序数据清洗

① 特征提取策略

将长时序数据（645万行）降维为每个患者的统计特征

避免直接使用时序原始数据（会导致特征维度过高）

② 选择的统计量

均值、标准差：反映整体水平和波动性

最小值、最大值：捕捉极端情况

根据临床意义选择：

心率、血压：关注波动（用std）

血氧、GCS：关注最低值（病情恶化）

SOFA评分：关注最大值（器官衰竭严重程度）

呼吸机使用：关注使用比例（mean）

③ 为何不插值填充缺失值

采用前向填充（ffill）后直接提取统计特征

避免过多人为插值引入偏差

四、设计权衡
1. 异常值处理：选择删除极端值，理由是避免模型被异常值误导
2. 缺失值填充：选择保留样本，理由是样本数本身有限（7.6万→最终约7万）
3. 时序数据：选择提取统计特征，理由是避免维度爆炸，保留关键信息
4. 文本数据：选择保留原始文本，理由是为深度学习模型保留信息
5. 特征选择：选择暂不选择，理由是让后续模型自动学习特征重要性
